Session variables are usefull to store global variables to access throughout the application lifecycle opening and closing. Session variables store stored in server and can access in more than one web page request. For example session can use to store logged in user details for a web application.

CodeIgniter framework has session library to manage PHP global array $_SESSION.

Initialize the session in controller when using CodeIgniter. 
		          	
					$this->load->library('session');  // initialize the class
					// after that use '$this->session' to access loaded session.
Set a value to session array. 
		          	
					$_SESSION['item'] = 'item_value';     // using $_SESSION array
					$this->session->set_userdata('item', 'item_value');   //using session class setter
Get a value from session array. 
		          	
					$_SESSION['item'];        // using $_SESSION array
					$this->session->userdata('item');  //using session class getter
Check value is exist in session array 
		          	
					isset($_SESSION['item']);        // using $_SESSION array
					$this->session->has_userdata('item');  //using session class method 'has_userdata'
Remove data in session array. 
		          	
					unset($_SESSION['item']);       
					$this->session->unset_userdata('item');
Destroy the session. This delets all flash and temp data. 
		          	
					$this->session->sess_destroy();
Flash Data

Specific session data type which are only available for next request.

		          	
					$this->session->mark_as_flash('item');   // set flash data
					$this->session->flashdata('item');       // read flash data
Temp Data

Specific session data type which has expiration time.

		          	
					$this->session->set_tempdata('item', 'value', 300);  // add temp data with 300 seconds expiry time
'Account_controller' inside the Student Application has a example of this session data usage for login management
